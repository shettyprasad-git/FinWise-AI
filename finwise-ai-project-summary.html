Bug Fixes Applied (this session)

1
JS race condition — nothing worked on any page
All page scripts destructured window.FW before main.js had set it, crashing silently. Fixed by: (a) moving window.FW = ... to the very first statement in main.js, (b) wrapping all page-specific scripts in DOMContentLoaded.
2
Wrong IBM API key — service credential vs personal key
The auto-generated service credential Xi7sUN… cannot call watsonx.ai generation. Replaced with personal IAM key Vpf03H… which has full access.
3
Truncated instance UUID — watsonx Orchestrate returning 500
The URL was missing -f0bea2a0cc1f from the instance UUID. Corrected to the full 5d0cb4d3-24cd-4940-80c2-f0bea2a0cc1f.
4
Wrong AI endpoint — Granite model not available in au-syd
Switched from the deprecated ibm/granite-13b-chat-v2 (unavailable in au-syd) + /ml/v1/text/generation to meta-llama/llama-3-3-70b-instruct + /ml/v1/text/chat (chat-completions format, confirmed live).
5
Missing WATSONX_AI_URL in .env
The env file was missing WATSONX_AI_URL, causing the app to silently fall back to a hardcoded default. Added explicitly to .env.
